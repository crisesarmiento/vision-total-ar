# Security Policy

## Supported Versions

Security fixes are applied to the current production branch (`main`) and the active integration branch (`develop`). Older snapshots and stale feature branches are not supported.

## Reporting a Vulnerability

Do not open a public GitHub issue for a suspected vulnerability.

Preferred path:
1. Use GitHub private vulnerability reporting for this repository if it is enabled.
2. If private reporting is not enabled yet, contact the maintainer privately using the contact method listed on the GitHub profile before public disclosure.

Please include:
- a concise description of the issue
- affected routes, pages, or APIs
- reproduction steps or proof of concept
- impact assessment
- any suggested mitigation if you have one

## Disclosure Expectations

- Give maintainers reasonable time to validate and remediate the issue before public disclosure.
- Keep exploit details private until a fix or mitigation is available.
- If a fix requires follow-up hardening work, maintainers may track that in Linear after the immediate vulnerability is addressed.
