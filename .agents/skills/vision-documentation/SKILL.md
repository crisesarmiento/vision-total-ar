# Vision Documentation

Use this skill for public docs, runbooks, release notes, PR bodies, and Linear issue descriptions.

## Style

- Be concise, operational, and public-safe.
- Prefer concrete commands and file paths when useful.
- Explain what changed, why it matters, and how to validate it.
- Keep public docs free of secrets, private URLs, personal preferences, and unreleased vulnerability detail.

## Common Updates

- README: setup, commands, architecture, release flow, and integration overview.
- Runbooks: production-safe operational workflows with explicit verification and rollback.
- PR descriptions: Linear issue, summary, checklist, validation, preview or N/A.
- Linear tickets: problem, root cause or goal, implementation plan, acceptance criteria, priority, estimate, labels, cycle, milestone.

If a doc would need private details, write the public boundary and point maintainers to private skills or private systems.
